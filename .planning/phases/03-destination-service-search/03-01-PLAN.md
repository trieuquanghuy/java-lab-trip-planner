    **CityRepository.java** — Spring Data JPA repository with native query for population-weighted FTS:
    ```java
    public interface CityRepository extends JpaRepository<City, Long> {

        @Query(value = """
            SELECT c.* FROM destination.cities c,
                   to_tsquery('simple', unaccent(trim(:query)) || ':*') AS q
            WHERE c.search_tsv @@ q
            ORDER BY ts_rank(c.search_tsv, q) * LOG(c.population + 1) DESC
            LIMIT :limit
            """, nativeQuery = true)
        List<City> searchByPrefix(@Param("query") String query, @Param("limit") int limit);
    }
    ```
    Key details per CONTEXT.md decisions:
    - D-04: Population-weighted ranking `ts_rank * LOG(population + 1) DESC`
    - D-05: Prefix matching via `|| ':*'` suffix on query
    - D-06: `'simple'` text search config (no stemming for proper nouns)
    - D-07: `unaccent()` on query input for accent folding
    - The native query uses `to_tsquery` (not `plainto_tsquery`) to support the `:*` prefix operator
    - Schema-qualify the table as `destination.cities` in the native query to avoid search_path ambiguity
  </action>
