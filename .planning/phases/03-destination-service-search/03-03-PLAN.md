    ```java
    @Service
    public class SearchService {
        private static final int MAX_LIMIT = 5;          // D-14: cap at 5 regardless of request
        private static final Duration CACHE_TTL = Duration.ofHours(1);  // D-09: 1h TTL
        private static final Duration LOCK_TTL = Duration.ofSeconds(5); // D-10: 5s lock
        private static final String CACHE_PREFIX = "SEARCH:";           // D-08
        private static final String LOCK_PREFIX = "lock:search:";       // D-10

        private final CityRepository cityRepository;
        private final StringRedisTemplate redisTemplate;                // D-11: String template
        private final ObjectMapper objectMapper;

        // Constructor injection (all 3 deps)
    }
    ```

    **`search(String q, String type, int limit)` method:**

    1. **Input normalization:** `q = q == null ? "" : q.trim()`. If `q` is empty or blank, return `SearchResponse.empty()` immediately (D-12).

    2. **Limit capping:** `limit = Math.min(Math.max(limit, 1), MAX_LIMIT)` — D-14.

    3. **Cache key:** `String cacheKey = CACHE_PREFIX + q.toLowerCase() + ":" + type + ":" + limit` — D-08, normalize to lowercase.

    4. **Cache read:** `String cached = redisTemplate.opsForValue().get(cacheKey)`. If non-null, deserialize with ObjectMapper and return.

    5. **Single-flight lock (D-10, Pitfall 8):**
       ```java
       String lockKey = LOCK_PREFIX + cacheKey;
       Boolean acquired = redisTemplate.opsForValue()
           .setIfAbsent(lockKey, "1", LOCK_TTL);
       if (Boolean.TRUE.equals(acquired)) {
           try {
               // Double-check cache after acquiring lock
               cached = redisTemplate.opsForValue().get(cacheKey);
               if (cached != null) {
                   return objectMapper.readValue(cached, SearchResponse.class);
               }
               // Query Postgres
               List<City> cities = cityRepository.searchByPrefix(q, limit);
               SearchResponse response = toResponse(cities);
               // Write to cache
               redisTemplate.opsForValue().set(cacheKey,
                   objectMapper.writeValueAsString(response), CACHE_TTL);
               return response;
           } finally {
               redisTemplate.delete(lockKey);
           }
       } else {
           // Another thread holds the lock — brief retry then query directly
           // Wait up to 500ms for cache to be populated
           for (int i = 0; i < 5; i++) {
               Thread.sleep(100);
               cached = redisTemplate.opsForValue().get(cacheKey);
               if (cached != null) {
                   return objectMapper.readValue(cached, SearchResponse.class);
               }
           }
           // Fallback: query Postgres directly (don't block forever)
           List<City> cities = cityRepository.searchByPrefix(q, limit);
           return toResponse(cities);
       }
       ```

    6. **`toResponse(List<City> cities)` private method:**
       Map each City to CitySearchItem:
       ```java
       private SearchResponse toResponse(List<City> cities) {
           List<CitySearchItem> items = cities.stream()
               .map(c -> new CitySearchItem("city", c.getName(), c.getCountry(), c.getLat(), c.getLng()))
               .toList();
           return new SearchResponse(items);
       }
       ```
       Per D-13, the `type` field is "city" for all results from the cities table. Country-type results are also sourced from cities table (countries appear as city entries with the country name).

    Handle `InterruptedException` from `Thread.sleep` by restoring the interrupt flag and falling through to the Postgres query.

    Handle Jackson `JsonProcessingException` in cache read by logging a warning and falling through to Postgres query (cache corruption should not break search).

    Use `@Slf4j` (Lombok) or create a `private static final Logger log = LoggerFactory.getLogger(SearchService.class)` for logging cache hits/misses.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | T (Tampering) | SearchController query params | mitigate | `q` is trimmed and normalized before use. `limit` is capped to [1,5] range. `type` is used only as a pass-through filter string (not in SQL). Spring's `@RequestParam` handles URL decoding safely. |
| T-03-07 | D (Denial of Service) | Single-flight lock | mitigate | Lock TTL capped at 5 seconds (auto-expires). Failed lock acquisition falls through to direct Postgres query (no indefinite blocking). Cache TTL 1h prevents repeated cold queries. |
| T-03-08 | S (Spoofing) | Public endpoint | accept | Search is intentionally public (no auth required). Rate limiting enforced at gateway level (Phase 1 D-06: 60 req/min per IP). No sensitive data exposed. |
| T-03-09 | T (Tampering) | Redis cache key injection | mitigate | Cache key is constructed from normalized lowercase input. Redis key format `SEARCH:{q}:{type}:{limit}` — colons are Redis key separators but pose no injection risk (Redis keys are opaque strings, not interpreted). No EVAL or Lua scripts used for cache read/write. |
