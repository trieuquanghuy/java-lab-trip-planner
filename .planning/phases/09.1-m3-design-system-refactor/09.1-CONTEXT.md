# Phase 9.1: M3 Design System Refactor - Context

**Gathered:** 2026-05-19
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase replaces the default shadcn/ui slate palette and generic tokens with a Material Design 3 tonal color system, typography scale, shape tokens, and motion system — all applied via CSS variables and Tailwind config. No library swaps, no component API changes, no new features.

</domain>

<decisions>
## Implementation Decisions

### Color Mapping Strategy
- **D-01:** Keep all existing shadcn CSS variable names (`--primary`, `--muted`, `--card`, etc.) — change their HSL values to M3 tonal palette equivalents. Components reference these variables already; no component code changes needed.
- **D-02:** Add new CSS variables for M3 roles that have no shadcn equivalent: `--primary-container`, `--on-primary-container`, `--secondary-container`, `--on-secondary-container`, `--surface-tint`, `--outline`, `--outline-variant`. These enable future M3-aware components without blocking this phase.
- **D-03:** Variable mapping (shadcn → M3 role):
  - `--primary` → M3 primary
  - `--primary-foreground` → M3 on-primary
  - `--secondary` → M3 secondary
  - `--secondary-foreground` → M3 on-secondary
  - `--muted` → M3 surface-variant
  - `--muted-foreground` → M3 on-surface-variant
  - `--accent` → M3 secondary-container
  - `--accent-foreground` → M3 on-secondary-container
  - `--card` → M3 surface
  - `--card-foreground` → M3 on-surface
  - `--background` → M3 surface (lightest)
  - `--foreground` → M3 on-surface
  - `--border` → M3 outline-variant
  - `--input` → M3 outline-variant
  - `--ring` → M3 primary
  - `--destructive` → M3 error
  - `--destructive-foreground` → M3 on-error
  - `--popover` → M3 surface-container
  - `--popover-foreground` → M3 on-surface

### Dark Mode Toggle
- **D-04:** System preference auto-detect on first visit (`prefers-color-scheme` media query) + manual override toggle in the navigation header.
- **D-05:** Toggle icon: sun/moon icon button using Lucide icons (already in project). Positioned in the Layout header, right side.
- **D-06:** Persist user preference to `localStorage` key `theme`. Values: `"light"`, `"dark"`, `"system"`. Default: `"system"`.
- **D-07:** Implementation: inline `<script>` in `index.html` head to apply `.dark` class before React hydrates (prevents flash of wrong theme).

### Palette Generation Method
- **D-08:** Use the M3 theme builder (https://m3.material.io/theme-builder) to generate tonal palette from seed `#1A73E8`, then hardcode the resulting HSL values directly in `index.css`. No runtime dependency on `@material/material-color-utilities`.
- **D-09:** Rationale: seed color is static (won't change at runtime), keeps bundle lean, no build-time generation complexity. If seed changes in future, regenerate manually.

### Typography Scale
- **D-10:** Map only the 7 M3 type variants actually used in the app:
  - `text-headline-lg` — page titles (32px/40px, weight 400)
  - `text-title-lg` — section headers (22px/28px, weight 400)
  - `text-title-md` — card titles (16px/24px, weight 500)
  - `text-body-lg` — descriptions (16px/24px, weight 400)
  - `text-body-md` — regular text (14px/20px, weight 400)
  - `text-label-lg` — buttons, nav items (14px/20px, weight 500)
  - `text-label-sm` — metadata, timestamps (11px/16px, weight 500)
- **D-11:** Define as Tailwind theme extension utilities (not @apply classes). Font family stays system sans-serif (no Google Fonts import).

### Shape Scale
- **D-12:** Replace current `--radius: 0.5rem` with M3 shape scale tokens:
  - `--radius-none`: 0px
  - `--radius-xs`: 4px
  - `--radius-sm`: 8px
  - `--radius-md`: 12px
  - `--radius-lg`: 16px
  - `--radius-xl`: 28px
  - `--radius-full`: 9999px
- **D-13:** Keep `--radius` as alias to `--radius-md` (12px) for backward compat with existing shadcn components that reference `var(--radius)`.

### Elevation (Surface Tint)
- **D-14:** Replace box-shadow elevation with M3 surface-tint approach: elevated surfaces get progressively mixed with `--surface-tint` color (primary at low opacity). Define CSS variables:
  - `--elevation-1`: surface-tint at 5% opacity
  - `--elevation-2`: surface-tint at 8% opacity
  - `--elevation-3`: surface-tint at 11% opacity
- **D-15:** Apply via Tailwind utilities: `bg-elevation-1`, `bg-elevation-2`, `bg-elevation-3`. Cards use elevation-1, popovers use elevation-2, modals use elevation-3.

### Motion Tokens
- **D-16:** Replace existing fixed `ease-out 0.4s` with M3 motion tokens:
  - `--duration-short`: 150ms (micro-interactions, icons)
  - `--duration-medium`: 300ms (page transitions, cards)
  - `--duration-long`: 500ms (complex animations, route changes)
  - `--easing-standard`: cubic-bezier(0.2, 0, 0, 1) — standard M3
  - `--easing-emphasized`: cubic-bezier(0.2, 0, 0, 1) — emphasized entry
  - `--easing-emphasized-decelerate`: cubic-bezier(0.05, 0.7, 0.1, 1) — emphasized exit
- **D-17:** Update existing keyframe animations in `index.css` to use these tokens. Add `prefers-reduced-motion` media query to disable animations.

## Existing Code Insights

### Reusable Assets
- `tailwindcss-animate` plugin — already installed, provides animation utilities that can be extended with M3 motion tokens
- `components/ui/` — All shadcn components reference CSS variables via `hsl(var(--name))` pattern — zero component changes needed for color swap
- Lucide React icons — already in project for the dark mode toggle icon

### Established Patterns
- Colors: All via `hsl(var(--name))` in Tailwind config → CSS variables in index.css
- Dark mode: `.dark` class on `<html>`, toggled via Tailwind's `darkMode: 'class'`
- Border radius: `var(--radius)` with calc-based `md` and `sm` variants
- Animations: Custom keyframes in `@layer utilities` + `@keyframes` blocks

### Integration Points
- `frontend/src/index.css` — Replace `:root` and `.dark` variable blocks (primary change)
- `frontend/tailwind.config.ts` — Extend with typography utilities, shape tokens, elevation utilities
- `frontend/index.html` — Add inline theme detection script in `<head>`
- `frontend/src/components/Layout.tsx` — Add dark mode toggle button
