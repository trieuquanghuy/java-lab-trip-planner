<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet/LAN ↔ gateway :8080 | Gateway is the public surface; everything else is loopback-only (D-22) |
| gateway ↔ downstream services on compose network | Static-URI routing; no auth between containers in Phase 0 (Phase 1 adds JWT) |
| gateway ↔ Eureka :8761 | Lease registration; trusted within compose network |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-00-19 | Spoofing | A request bypasses the gateway and hits downstream service ports directly with a forged X-User-Id (Pitfall 1 attack mode) | HIGH | mitigate (partial) | D-22 binds downstream services to `127.0.0.1` only — neuters the LAN attack vector. Phase 1 ships JwtCommonFilter to enforce JWT validation server-side regardless of where the request came from. Phase 0 has no JWT validation yet, but no domain endpoints either, so the attack surface is empty. |
| T-00-20 | Information Disclosure | Actuator endpoints leak sensitive config (`/actuator/env`, `/actuator/configprops`) | HIGH | mitigate | `management.endpoints.web.exposure.include=health,info,prometheus` ONLY (Task 6.2). `env`, `configprops`, `beans`, `mappings` are NOT exposed. |
| T-00-21 | DoS | Gateway exposed on `0.0.0.0:8080` accepts unlimited requests | MEDIUM | accept (Phase 1 owns) | Phase 1 ships rate limiting at the gateway (5 req/15min for /api/auth/login per IP+email). Phase 0 has no real endpoints to rate-limit; `/__health/<svc>` are intentionally unauthenticated for ops monitoring. |
| T-00-22 | Tampering | Static-URI routing can be re-pointed by editing `application.yml` to a malicious host | LOW | accept | Source-control review; configuration-as-code is the threat model assumption. |
| T-00-23 | Information Disclosure | `show-details: always` on `/actuator/health` reveals component status (DB up/down, etc.) | LOW | accept | Phase 0 gateway has no DB; no sensitive components in the health response. Phase 2+ may switch to `when-authorized` if real DB info is added. |
</threat_model>
