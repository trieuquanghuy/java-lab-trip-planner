  <action>
    Create DTOs in package `com.tripplanner.destination.search` matching docs/04-api-spec.md §4:

    **CitySearchItem.java** — Record representing a single search result:
    ```java
    public record CitySearchItem(
        String type,     // "city" or "country"
        String name,
        String country,
        BigDecimal lat,
        BigDecimal lng
    ) {}
    ```
    Use `java.math.BigDecimal` for lat/lng to match the JPA entity. The `type` field is derived at the service layer (per D-13, the cities table stores both city and country searchable data; the type is determined by the query context).

    **SearchResponse.java** — Wrapper record:
    ```java
    public record SearchResponse(
        List<CitySearchItem> items
    ) {
        public static SearchResponse empty() {
            return new SearchResponse(List.of());
        }
    }
    ```
    The `empty()` factory method supports D-12 (empty/short query → return `{"items": []}`).

    Use Java records (not classes with getters) for immutability and compact serialization. Jackson serializes records correctly with Spring Boot's auto-configured ObjectMapper.
  </action>
