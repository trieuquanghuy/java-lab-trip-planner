| Browser SPA → api-gateway (`localhost:5173` → `:8080`) | The CORS-enforced public boundary. Any `Authorization` header here is untrusted until JWT signature verification passes. CORS allowlist (`localhost:5173` only with `allowCredentials=true`) is the first line of defense. |

| api-gateway → trip-service / auth-service / destination-service (compose-internal) | Within compose, service-to-service traffic is unencrypted but trusted by network topology. | Redis (`redis:6379`) → api-gateway | Rate-limit state crosses the bridge. Redis container is compose-private (no public port). State leakage / poisoning would require compromising the bridge network — out of scope for v1. Keyspace prefix `rl:*` is documented (CONTEXT.md `<specifics>`). |
